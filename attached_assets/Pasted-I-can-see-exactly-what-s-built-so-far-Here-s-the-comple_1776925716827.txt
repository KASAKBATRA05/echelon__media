## SCROLL ANIMATIONS — GLOBAL SYSTEM (CRITICAL — THIS IS WHAT'S MISSING)

This entire website must feel alive while scrolling. Use GSAP + ScrollTrigger (CDN) for all scroll effects. Every single section must have scroll-driven animation. Nothing is static.

**Required global scroll behaviors:**

1. **Fade + Rise on enter:** Every section's content starts at `opacity: 0, y: 60` and animates to `opacity: 1, y: 0` as it enters the viewport. Duration `0.8s`, ease `power3.out`. Children stagger `0.12s` apart.

2. **Horizontal slide-in for alternating elements:** Left-side elements enter from `x: -80`, right-side elements from `x: 80`. Triggered by ScrollTrigger with `start: "top 80%"`.

3. **Scale reveal for images:** All images start at `scale: 1.1, opacity: 0` and animate to `scale: 1, opacity: 1` on scroll entry, `duration: 1s`. This creates a "zooming into place" cinematic feel.

4. **Pinned scroll sections:** The "Our Work Speaks" portfolio section uses `ScrollTrigger pin: true` — the section title stays pinned while project cards scroll past it horizontally.

5. **Number counters:** Any stat numbers (e.g., "200+ Projects") count up from 0 when scrolled into view using GSAP `TextPlugin` or a manual counter loop.

6. **Parallax on section backgrounds:** Each section's background moves at `0.4x` scroll speed relative to content, creating depth. Use `ScrollTrigger scrub: 1` on a `backgroundPositionY` or `y` transform on a pseudo-element.

7. **Golden line draw:** Between sections, a thin golden horizontal rule `1px solid #F5A623` draws from left to right using `scaleX: 0 → 1, transformOrigin: left` triggered on scroll.

8. **Text scramble on headings:** Section headings scramble through random characters and resolve into the real text when scrolled into view. Use a JS character scramble loop (30ms interval, 600ms total).
